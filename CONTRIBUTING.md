# Contributing

## 開発環境

```bash
npm install
npm run dev
```

本番ビルド確認：

```bash
npm run build
```

## ブランチ

- `main`：公開ブランチ
- `docs/<topic>`：文書追加・修正
- `feat/<topic>`：サイト機能追加
- `fix/<topic>`：不具合修正

1 Issue、1 Branch、1 Pull Requestを原則とする。

## ページ構成

各実践ページには可能な範囲で次を含める。

1. このページで分かること
2. 前提環境
3. 背景と仕組み
4. 実行手順
5. 期待される結果
6. 内部で起きていること
7. よくあるエラー
8. 次に読むページ
9. 参考資料

## 表記

- 製品名は`ROS 2`と表記
- コマンドには対象環境を明記
- 未検証手順を検証済みと表現しない
- 公式文章を転載せず、要約と参照リンクを使用
